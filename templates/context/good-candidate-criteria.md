# Good Candidate Criteria

A repo is interesting to Vega Lab when it has one or more of these signals:

- Teaches an implementation pattern useful to core-x houses.
- Provides agent, MCP, workflow, local model, or repo-ops capability.
- Can become a reusable template or scaffold after cleanup.
- Has strong adoption evidence: active maintenance, clear license, useful docs, and focused scope.
- Exposes a weak spot in owned repositories that Vega Lab can turn into a rule, skill, workflow, or automation.
